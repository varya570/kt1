Техническое задание (ТЗ)

1. Введение

Данный документ описывает требования к Telegram-боту для местной кофейни.

2. Цели и задачи

Цель проекта

Создать Telegram-бота, который позволит пользователям оформлять заказ напитков и получать их удобным способом.

Задачи
 • Отображать пользователю меню с напитками.
 • Позволять выбирать один или несколько товаров.
 • Реализовать оплату через QR-код.
 • Отдавать пользователю подтверждение об успешной оплате.
 • Предоставлять выбор формата получения заказа: самовывоз или доставка.

3. Требования к функционалу

Основной функционал:
 1. Меню
 • Позиции меню содержат название и цену.
 2. Формирование заказа
 • Пользователь может выбрать одну или несколько позиций.
 • Подсчёт итоговой стоимости производится автоматически.
 3. Оплата
 • После выбора товаров бот генерирует QR-код для оплаты.
 • Пользователь получает уведомление об успешной оплате.
 4. Получение заказа
 • Два варианта:
• «Заберу сам»
• «Доставка до офиса»
 • Если выбран вариант «доставка», пользователь должен ввести полный адрес или описание точки доставки.
 5. Уведомление администратора
 • После оформления заказа данные отправляются администратору кофейни.

4. Требования к интерфейсу
• Интерфейс — Telegram-чат с кнопками.
• Основные элементы:
• Кнопка «Меню»
• Кнопки для выбора позиций
• Кнопки выбора формата получения заказа
• QR-код предоставляется в виде изображения.

5. Нефункциональные требования
 • Бот должен корректно работать на мобильных устройствах.
 • Доступность 24/7.
 • Корректная обработка ошибок ввода (например, некорректный адрес при доставке).

6. Интеграции и окружение
 • Интеграция с Telegram Bot API.
 • Генерация QR-кода через внешний сервис или библиотеку.
 • Доступ администратора — через отдельный Telegram-аккаунт или чат.

7. Ограничения и допущения
 • Бот не обрабатывает оплату внутри Telegram — только предоставляет QR-код.
 • Меню обновляется вручную администратором.
 • Предполагается стабильное интернет-соединение пользователя.

8. Критерии приёмки

Работа считается выполненной, если:
 • Бот показывает меню.
 • Можно выбрать несколько позиций.
 • Корректно формируется итог заказа.
 • Генерируется QR-код.
 • Пользователь получает подтверждение оплаты.
 • Реализован выбор: «заберу сам» / «доставка».
 • Администратор получает уведомление о новом заказе.
